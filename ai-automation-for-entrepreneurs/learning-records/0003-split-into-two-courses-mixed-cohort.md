# Program split into two stackable courses; mixed-cohort design

The instructor reframed Part 2 and grew the scope significantly. Two decisions set the
structure going forward:

1. **Two courses, not one.** Course A "Automate & Build" (foundational) + Course B "Ship
   Real Software with AI Agents" (advanced). Stackable; A doesn't scare off non-technical
   founders, B lets the ambitious go pro.
2. **Mixed cohort.** Every module = core lessons (everyone) + `◆` deep-dive lessons
   (optional, for the ambitious). Build core first; deep-dives on demand.

Instructor's content corrections that drove this (important — don't revert):
- The "final 20%" is NOT "reading code / debugging." It's **production reality**: GitHub
  & worktrees, databases, hosting/infra, API keys & secrets, auth, payments. This is now
  Course B Module 1. (Reading-code/debugging demoted to lightweight lessons inside Course A's vibe-coding module.)
- Add an **agentic toolkit** module: anatomy of an agent, tools/MCP/skills, subagents,
  session logs, open source. (Course B Module 2.)
- The intro must **break down the AI tool landscape categorically** (a map, not a list) —
  Course A, lesson A1.2.
- Add **anatomy of an app** (Course A, A4.2) and **anatomy of an agent** (Course B, B2.1)
  as orienting "map" lessons.

Implication: audience has shifted from "non-technical entrepreneur" toward "aspiring
technical founder" for Course B. Course A stays accessible. Built lessons 0001 and 0013
both belong to Course A.
