# Mission expanded: from "automate" to "automate AND build" (vibe coding)

The instructor broadened the mission. It's no longer only no-code workflow automation —
learners should also become **technical enough to feel empowered to build their own
stuff**: part no-code, part true code ("vibe coding"). Confirmed via direct request,
so MISSION.md was updated and this record logged (per the skill's mission-shift rule).

Key parameters set with the instructor:
- **Build target:** "a bit of everything" — automations, internal tools/scripts, small apps. Broad empowerment, not one narrow artifact.
- **Code depth:** "just enough to not get stuck" — vibe-first, but learn to *read* generated code, *debug*, and use *Git/deploy* enough to escape the "final 20% / comprehension-debt" trap. Empowerment over mastery.
- **Scope:** right-sized to **6 weeks** — 3-week automation core + 3-week build half, ~18 lessons + 6 live sessions.

Why this steers future sessions:
- A whole module (Module 5) now exists specifically to counter the documented "vibe-coding hangover": founders who generate 80% of an app then get stuck, burn credits debugging, and own a codebase they can't read. Lessons must build *enough* comprehension to escape this — and no more.
- Design against the "blank chat box" UX trap: teach learners to build *results/buttons*, not chat interfaces (sourced pitfall).
- Tooling now spans two stacks: automation (Zapier/Make) + vibe coding (Lovable/Bolt to build, Replit/Cursor to graduate). Anchor decisions in NOTES.md.
- Capstone is now learner's choice: an automation OR a small app.
