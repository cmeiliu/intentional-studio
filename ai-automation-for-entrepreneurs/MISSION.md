# Mission: AI for Non-Technical Entrepreneurs — Automate & Build

## Why
Non-technical founders are boxed in twice: they lose hours to repetitive busywork,
*and* every idea for a tool or app has to wait on someone else to build it. This class
removes both walls. By the end, each entrepreneur can **automate a real workflow** in
their business **and build a working tool or app themselves** with AI — and understands
the code well enough to fix it when it breaks, instead of being trapped owning software
they can't read.

## Success looks like
- A learner can say, in plain language, where AI does and doesn't belong in their business.
- A learner can write a reliable, reusable prompt for a recurring task.
- A learner can build a working no-code automation (trigger → AI action → result).
- A learner can **build and deploy a small working app/tool by describing it** ("vibe coding") — with a real, public URL.
- A learner can **read the code an AI generated, debug an error, and use Git/deploy** well enough to escape the "final 20%" trap — not fluent, but unstuck.
- Each learner **ships one capstone** — an automation or a small app — running for real by course end.

## Constraints
- **Audience:** non-technical — no prior coding. Comfortable with everyday SaaS.
- **Time:** busy founders. Target ~3–4 hrs/week, **6 weeks**. Lessons stay 10–15 min.
- **Delivery:** hybrid — self-paced HTML lessons + one live build/Q&A session per week.
- **Budget:** free / low-cost tiers where possible; flag when a paid tier is needed (AI build credits especially).
- **Code depth is bounded:** "just enough to not get stuck." Empowerment over mastery — read, debug, and ship, not computer-science fundamentals.

## In scope (newly added)
- "Vibe coding": building software by describing it to an AI (Lovable / Bolt / v0 / Replit).
- Reading generated code, debugging with AI, basic Git, and deploying.
- Knowing your limits — when a build is good enough vs. when to bring in a real developer.

## Out of scope (for this course)
- Computer-science fundamentals, algorithms, writing production code from a blank file.
- Backend architecture, training/fine-tuning models, self-hosting infrastructure.
- Becoming a software engineer. The goal is an empowered builder, not a professional dev.
